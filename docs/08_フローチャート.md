## 1. 認証フロー

```mermaid
flowchart TD
    A([アプリ起動]) --> B[supabase.auth.getSession]
    B --> C{セッションあり?}
    C -->|No| D[LoginPage表示]
    C -->|Yes| E[AppLayout表示]

    D --> F{ログイン方法選択}
    F -->|Google| G[signInWithOAuth\ncalendar scope]
    F -->|メール\nDEV only| H[signInWithPassword]

    G --> I[Googleアカウント選択]
    I --> J[カレンダー権限許可]
    J --> K[Supabase Callback\nprovider_token取得]
    K --> E

    H --> L{認証成功?}
    L -->|No| M[エラーメッセージ表示]
    L -->|Yes| E
    M --> D

    E --> N[loadMasters / loadAppTasks\n/ loadBacklogToken 並列実行]
    N --> O([メイン画面 Ready])
```

## 2. 今日タブ初期化フロー

```mermaid
flowchart TD
    A([TodayView マウント]) --> B[devDate or 今日の日付を決定]
    B --> C[fetchTodayEvents\nGoogle Calendar API]
    C --> D{取得成功?}
    D -->|Error 401| E[再ログインを促す]
    D -->|Success| F[app_records から\ntarget_dateのレコードを検索]

    F --> G{レコードあり?}
    G -->|No| H[app_records に INSERT]
    G -->|Yes| I[既存レコードIDを使用]
    H --> I

    I --> J[app_record_details から\nrecord_idで明細取得]
    J --> K[カレンダー予定と明細をマージ\ncalendar_event_idで結合]

    K --> L[未紐付けイベントに\nautoLink実行]
    L --> M{confidence = high?}
    M -->|Yes| N[task_idを自動設定\nautoLinked=true]
    M -->|No| O[task_id=null のまま]
    N --> P[todayEventsをストアにセット]
    O --> P
    P --> Q([画面表示])
```

## 3. タイマー操作フロー

### 3-1. タイマー開始

```mermaid
flowchart TD
    A([「開始」ボタンクリック]) --> B{既存の\ndetailIdあり?}
    B -->|No| C[app_record_details に\nINSERT\nactual_start=now]
    B -->|Yes| D[app_record_details を\nUPDATE\nactual_start=now\npause_log=clear]

    C --> E[detailIdをイベントにセット]
    D --> E

    E --> F[activeEventId = eventId\nisPaused = false\npausedAt = null]
    F --> G[1秒ごとにre-render開始]
    G --> H([TimerHero でカウントダウン表示])
```

### 3-2. 一時停止・再開

```mermaid
flowchart TD
    A([「一時停止」クリック]) --> B{現在の状態}
    B -->|isPaused=false\n進行中| C[isPaused = true\npausedAt = now]
    B -->|isPaused=true\n停止中| D[pause_logに\nes: pausedAt, e: now を追加]

    C --> E[DB: pause_log更新\nactual_end=nullのまま]
    D --> F[isPaused = false\npausedAt = null]
    F --> G[DB: pause_log更新]

    E --> H([一時停止表示・黄色])
    G --> I([再開表示・青/赤])
```

### 3-3. タイマー終了

```mermaid
flowchart TD
    A([「終了」ボタンクリック]) --> B{isPaused = true?}
    B -->|Yes| C[pause_logを閉じる\ne: now を設定]
    B -->|No| D[pause_logそのまま]

    C --> E[app_record_details UPDATE\nactual_end=now\npause_log=最終化]
    D --> E

    E --> F[activeEventId = null\nisPaused = false]
    F --> G[タスクカードに\n実績時間表示]
    G --> H([status = done に変更])
```

### 3-4. 手動時間調整

```mermaid
flowchart TD
    A([スライダー操作]) --> B[override_elapsed_ms = 選択値ms]
    B --> C[app_record_details UPDATE\noverride_elapsed_ms]
    C --> D([TimerHeroに✎マーク表示])

    E([「リセット」クリック]) --> F[override_elapsed_ms = null]
    F --> G[app_record_details UPDATE\noverride_elapsed_ms=NULL]
    G --> H([自動計測に戻る・✎消える])
```

### 3-5. やり直し（Undo）

```mermaid
flowchart TD
    A([「↩ やり直し」クリック]) --> B[app_record_details UPDATE\nactual_start=NULL\nactual_end=NULL\npause_log=empty\noverride_elapsed_ms=NULL]
    B --> C[イベントのstatusを\npendingに変更]
    C --> D([タスクカードが未着手に戻る])
```

## 4. タスク紐付けフロー

### 4-1. LinkModal 表示

```mermaid
flowchart TD
    A([「未紐付け ＋」クリック]) --> B[LinkModal表示]
    B --> C[scoreCandidates実行\n全タスクをスコアリング]
    C --> D[score >= 0.35 のタスクを\n候補として表示]
    D --> E{タスク選択}

    E -->|既存タスク選択| F[確認ボタンクリック]
    E -->|「新規タスク」| G[TaskEditModal表示]
    E -->|「紐付け解除」| H[task_id = NULL に更新]

    G --> I[タスク作成後\n新タスクを選択]
    I --> F

    F --> J[app_record_details UPDATE\ntask_id=選択タスクID]
    J --> K[app_tasks UPDATE\nusage_count +1]
    K --> L[ストアのtodayEventsを更新]
    L --> M([タスクカードにタスクID表示])
```

### 4-2. 自動タスク紐付けスコアリング

```mermaid
flowchart TD
    A([カレンダー予定タイトル]) --> B[tokenize\n空白・記号で分割]
    B --> C[全タスクに対してループ]

    C --> D[タスクタイトルも tokenize]
    D --> E[共通トークン数を計算\noverlap]
    E --> F[titleScore = overlap /\nmax tokens 数]
    F --> G[historyBonus = min\nusage_count/20, 0.3]
    G --> H[score = min\ntitleScore + historyBonus, 1.0]

    H --> I{confidence判定}
    I -->|score >= 0.65| J[high - 自動紐付け]
    I -->|0.35 <= score < 0.65| K[medium - 提案のみ]
    I -->|score < 0.35| L[none - 提案なし]
```

## 5. Backlog連携フロー

### 5-1. 初回連携

```mermaid
flowchart TD
    A([「Backlog」ボタンクリック]) --> B[BacklogModal表示\n設定ビュー]
    B --> C[スペースキー入力]
    C --> D[「連携する」クリック]
    D --> E[sessionStorageに\nspace_keyを保存]
    E --> F[getAuthUrl生成\nstateはrandom UUID]
    F --> G[Backlog OAuth画面\nへリダイレクト]
    G --> H[ユーザー：Backlogでログイン\n・アプリを許可]
    H --> I[/backlog-callback\nへリダイレクト\n?code=xxx&state=xxx]
    I --> J[BacklogCallback.jsx]
    J --> K[POST /api/backlog-token\ngrant_type=authorization_code]
    K --> L[Vercel Serverless\nBacklog OAuth APIへ中継]
    L --> M{トークン取得成功?}
    M -->|No| N[エラー表示]
    M -->|Yes| O[backlog_tokens に\nUpsert]
    O --> P[ストアに\nbacklogTokenをセット]
    P --> Q[/ へリダイレクト]
    Q --> R([Backlogボタンが緑に])
```

### 5-2. 課題インポート

```mermaid
flowchart TD
    A([「課題をインポート」タブ]) --> B[ensureFreshToken\nトークン有効期限確認]
    B --> C{期限切れ寸前?}
    C -->|Yes| D[POST /api/backlog-token\ngrant_type=refresh_token]
    D --> E[backlog_tokens更新]
    C -->|No| F

    E --> F[GET /api/v2/users/myself\n自分のID取得]
    F --> G[GET /api/v2/issues\n担当課題取得\nstatusId=1,2,3]
    G --> H[既にインポート済みの課題を除外\nbacklog_issue_idで判定]
    H --> I[課題一覧表示]

    I --> J[ユーザー：インポートしたい\n課題をチェック]
    J --> K[「インポート」クリック]
    K --> L[各課題に対して\napp_tasks INSERT]
    L --> M[Backlogプロジェクト名と\nアプリ案件名を部分一致で紐付け]
    M --> N[ストアのappTasksに追加]
    N --> O([タスク管理タブに\nBバッジ付きで表示])
```

## 6. 集計データ取得フロー

```mermaid
flowchart TD
    A([集計タブ表示\nまたはフィルター変更]) --> B[期間・クライアント\nフィルターを確認]
    B --> C[sinceStr = 今日 - 選択期間]

    C --> D[Step1: app_records取得\nuser_id=自分\ntarget_date >= sinceStr]
    D --> E{recordsあり?}
    E -->|No| F[空の状態表示\n「記録がありません」]
    E -->|Yes| G[recordIds配列作成\nrecordMapを作成]

    G --> H[Step2: app_record_details取得\nrecord_id IN recordIds\nactual_end IS NOT NULL]
    H --> I[recordMapでenrich\n各detailにapp_recordsデータを追加]

    I --> J[クライアントフィルター適用]
    J --> K{アクティブなサブタブ}

    K -->|日別| L[target_dateでグループ化\n日ごとに集計]
    K -->|月別| M[YYYY-MMでグループ化\n月ごとに集計]
    K -->|タスク別| N[task_idでグループ化\nタスク名をストアから取得]
    K -->|案件別| O[project_idでグループ化\n案件名をストアから取得]
    K -->|カテゴリ別| P[category_idでグループ化\nカテゴリ名をストアから取得]

    L --> Q[各グループで集計\n予定min・実績min・差分・効率%]
    M --> Q
    N --> Q
    O --> Q
    P --> Q

    Q --> R([グラフ・テーブル表示])
```
