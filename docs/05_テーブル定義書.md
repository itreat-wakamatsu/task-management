## 1. テーブル一覧

| テーブル名 | 論理名 | 説明 | RLS |
|-----------|--------|------|-----|
| clients | クライアントマスター | 取引先企業・個人 | 参照：全員、更新：認証済み |
| projects | 案件マスター | クライアント配下のプロジェクト | 参照：全員、更新：認証済み |
| project_categories | カテゴリマスター | 案件配下の作業カテゴリ（2階層） | 参照：全員、更新：認証済み |
| app_tasks | タスク | ユーザーが管理するタスク | 本人のみ |
| app_records | 日次記録 | 日ごとの作業記録ヘッダー | 本人のみ |
| app_record_details | タイマー記録明細 | タイマー1回分の詳細記録 | 本人のみ |
| backlog_tokens | Backlogトークン | Backlog OAuthトークン | 本人のみ |

## 2. clients — クライアントマスター

| 列名 | 型 | NULL | デフォルト | 説明 |
|------|-----|------|-----------|------|
| id | bigint | NOT NULL | GENERATED BY DEFAULT | 主キー（自動採番） |
| name | varchar(255) | NOT NULL | - | クライアント正式名称 |
| display_name | varchar(255) | NULL | `''` | 表示名（省略表記など） |
| color | varchar(7) | NULL | `'#378ADD'` | UIカラー（#RRGGBB形式） |
| created_at | timestamptz | NOT NULL | `now()` | 作成日時 |
| deleted_at | timestamptz | NULL | NULL | 論理削除日時（NULL=有効） |

**主キー:** `id`

**サンプルデータ:**
```
id | name           | display_name | color
---+----------------+--------------+---------
 1 | 株式会社サンプル | サンプル社    | #4A90D9
 2 | フリーランス     | 個人          | #27AE60
```

## 3. projects — 案件マスター

|------|-----|------|-----------|------|
| id | bigint | NOT NULL | GENERATED BY DEFAULT | 主キー（自動採番） |
| client_id | bigint | NULL | NULL | クライアントID（FK: clients.id） |
| name | varchar(255) | NOT NULL | - | 案件名 |
| status | smallint | NOT NULL | `0` | 0: 進行中, 1: 完了 |

**主キー:** `id`
**外部キー:** `client_id → clients(id)`

**status 値定義:**

| 値 | 意味 |
|----|------|
| 0 | 進行中 |
| 1 | 完了 |

## 4. project_categories — カテゴリマスター

2階層のカテゴリ構造を自己参照で実現。

| 列名 | 型 | NULL | デフォルト | 説明 |
|------|-----|------|-----------|------|
| id | bigint | NOT NULL | GENERATED BY DEFAULT | 主キー |
| project_id | bigint | NULL | NULL | 案件ID（FK: projects.id） |
| parent_id | bigint | NULL | NULL | 親カテゴリID（FK: project_categories.id）。NULL=第1階層 |
| name | varchar(255) | NOT NULL | - | カテゴリ名 |
| order_no | int | NOT NULL | `1` | 表示順 |
| created_at | timestamptz | NOT NULL | `now()` | 作成日時 |
| deleted_at | timestamptz | NULL | NULL | 論理削除日時 |

**主キー:** `id`
**外部キー:**
- `project_id → projects(id)`
- `parent_id → project_categories(id)`

**階層ルール:**
- `parent_id IS NULL` → 第1階層（カテゴリ）
- `parent_id IS NOT NULL` → 第2階層（サブカテゴリ）

## 5. app_tasks — タスク

ユーザーが管理するタスクマスター。RLSにより本人のデータのみアクセス可。

| 列名 | 型 | NULL | デフォルト | 説明 |
|------|-----|------|-----------|------|
| id | bigint | NOT NULL | GENERATED BY DEFAULT | 主キー |
| user_id | uuid | NOT NULL | - | ユーザーID（FK: auth.users.id） |
| title | varchar(255) | NOT NULL | - | タスク名 |
| client_id | bigint | NULL | NULL | クライアントID（FK: clients.id） |
| project_id | bigint | NULL | NULL | 案件ID（FK: projects.id） |
| category_id | bigint | NULL | NULL | カテゴリID・第1階層（FK: project_categories.id） |
| subcategory_id | bigint | NULL | NULL | カテゴリID・第2階層（FK: project_categories.id） |
| status | smallint | NOT NULL | `0` | 0: 未着手, 1: 進行中, 2: 完了 |
| is_recurring | boolean | NOT NULL | `false` | 繰り返しタスクフラグ |
| usage_count | int | NOT NULL | `0` | タスク紐付け回数（自動提案スコアに利用） |
| start_date | date | NULL | NULL | 作業開始予定日 |
| due_date | date | NULL | NULL | 期日（YYYY-MM-DD） |
| backlog_issue_id | bigint | NULL | NULL | BacklogのIssue ID |
| backlog_issue_key | varchar(50) | NULL | NULL | BacklogのIssueキー（例: PROJ-42） |
| created_at | timestamptz | NOT NULL | `now()` | 作成日時 |
| updated_at | timestamptz | NOT NULL | `now()` | 更新日時 |
| deleted_at | timestamptz | NULL | NULL | 論理削除日時 |

**主キー:** `id`
**外部キー:**
- `user_id → auth.users(id)`
- `client_id → clients(id)`
- `project_id → projects(id)`
- `category_id → project_categories(id)`
- `subcategory_id → project_categories(id)`

**status 値定義:**

| 値 | 意味 | 画面表示 |
|----|------|---------|
| 0 | 未着手 | - |
| 1 | 進行中 | - |
| 2 | 完了 | - |

**期日超過判定:** `due_date < CURRENT_DATE AND status <> 2`

| user_id | uuid | NOT NULL | - | ユーザーID（FK: auth.users.id） |

## 7. app_record_details — タイマー記録明細

タイマー1回分の詳細記録。1つのカレンダー予定に対応。

| 列名 | 型 | NULL | デフォルト | 説明 |
|------|-----|------|-----------|------|
| id | bigint | NOT NULL | GENERATED BY DEFAULT | 主キー |
| record_id | bigint | NOT NULL | - | 日次記録ID（FK: app_records.id） |
| task_id | bigint | NULL | NULL | タスクID（FK: app_tasks.id）。紐付けなし=NULL |
| calendar_event_id | varchar(255) | NULL | NULL | GoogleカレンダーイベントID |
| calendar_event_title | varchar(255) | NULL | NULL | カレンダー予定タイトル（スナップショット） |
| planned_start | timestamptz | NULL | NULL | 計画開始日時（カレンダーより） |
| planned_end | timestamptz | NULL | NULL | 計画終了日時（カレンダーより） |
| actual_start | timestamptz | NULL | NULL | 実際の計測開始日時（タイマー開始時） |
| actual_end | timestamptz | NULL | NULL | 実際の計測終了日時（タイマー終了時） |
| override_elapsed_ms | int | NULL | NULL | 手動調整した実績時間（ミリ秒）。NULL=自動計算 |
| pause_log | jsonb | NOT NULL | `'[]'` | 一時停止ログ（配列）。構造は下記参照 |
| memo | text | NULL | NULL | メモ（現在UI未実装） |
| row_no | int | NOT NULL | `0` | 同一record内での表示順 |
| created_at | timestamptz | NOT NULL | `now()` | 作成日時 |
| updated_at | timestamptz | NOT NULL | `now()` | 更新日時 |

**主キー:** `id`
**外部キー:**
- `record_id → app_records(id)`
- `task_id → app_tasks(id)`

**pause_log の構造（JSONB配列）:**

```json
[
  { "s": "2026-04-04T10:15:00.000Z", "e": "2026-04-04T10:20:00.000Z" },
  { "s": "2026-04-04T10:45:00.000Z", "e": null }
]
```

| フィールド | 型 | 説明 |
|----------|-----|------|
| s | ISO 8601 string | 一時停止開始日時 |
| e | ISO 8601 string \| null | 一時停止終了日時（null=まだ停止中） |

**実績時間計算:**

```
実績時間 = override_elapsed_ms  // 手動調整値がある場合
         = actual_end - actual_start - sum(pause_log[i].e - pause_log[i].s)  // ない場合
```

## 8. backlog_tokens — Backlog OAuthトークン

ユーザーごとのBacklog OAuthトークン。

| 列名 | 型 | NULL | デフォルト | 説明 |
|------|-----|------|-----------|------|
| user_id | uuid | NOT NULL | - | 主キー（FK: auth.users.id） |
| access_token | text | NOT NULL | - | Backlog APIアクセストークン |
| refresh_token | text | NOT NULL | - | トークンリフレッシュ用 |
| expires_at | timestamptz | NOT NULL | - | アクセストークン有効期限 |
| space_key | varchar(100) | NOT NULL | - | Backlogスペースキー |
| created_at | timestamptz | NOT NULL | `now()` | 作成日時 |
| updated_at | timestamptz | NOT NULL | `now()` | 更新日時（トリガーで自動更新） |

**主キー:** `user_id`
**外部キー:** `user_id → auth.users(id)`

**トリガー:** `updated_at` は `backlog_tokens` 更新時に `now()` で自動セット

## 10. Row Level Security (RLS) 設定

### clients / projects / project_categories

```sql
-- SELECT: 全認証ユーザーが参照可
CREATE POLICY "Authenticated users can read"
  ON clients FOR SELECT
  TO authenticated
  USING (true);

-- INSERT/UPDATE: 認証済みユーザーが変更可
CREATE POLICY "Authenticated users can write"
  ON clients FOR INSERT, UPDATE
  TO authenticated
  WITH CHECK (true);
```

### app_tasks

```sql
CREATE POLICY "Users manage own tasks"
  ON app_tasks FOR ALL
  TO authenticated
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);
```

### app_records

```sql
CREATE POLICY "Users manage own records"
  ON app_records FOR ALL
  TO authenticated
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);
```

### app_record_details

```sql
-- record_id を通じて user_id を確認
CREATE POLICY "Users manage own record details"
  ON app_record_details FOR ALL
  TO authenticated
  USING (
    EXISTS (
      SELECT 1 FROM app_records
      WHERE app_records.id = app_record_details.record_id
        AND app_records.user_id = auth.uid()
    )
  );
```

### backlog_tokens

```sql
CREATE POLICY "Users manage own backlog tokens"
  ON backlog_tokens FOR ALL
  TO authenticated
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);
```
