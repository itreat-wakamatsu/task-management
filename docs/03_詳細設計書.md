## 2. 状態管理設計（Zustand）

### 2-1. ストア全体構造

```typescript
interface Store {
  // --- 認証 ---
  session: Session | null
  setSession: (session: Session | null) => void

  // --- マスターデータ ---
  clients: Client[]
  projects: Project[]
  categories: Category[]
  appTasks: Task[]
  setClients: (clients: Client[]) => void
  setProjects: (projects: Project[]) => void
  setCategories: (categories: Category[]) => void
  setAppTasks: (tasks: Task[]) => void

  // --- 今日タブ ---
  todayEvents: TodayEvent[]
  activeEventId: string | null
  isPaused: boolean
  pausedAt: Date | null
  devDate: Date | null

  setTodayEvents: (events: TodayEvent[]) => void
  setActiveEventId: (id: string | null) => void
  setIsPaused: (v: boolean) => void
  setPausedAt: (d: Date | null) => void
  setDevDate: (d: Date | null) => void
  updateEvent: (id: string, patch: Partial<TodayEvent>) => void
  addAppTask: (task: Task) => void

  // --- Backlog ---
  backlogToken: BacklogToken | null
  setBacklogToken: (token: BacklogToken | null) => void

  // --- 非同期アクション ---
  loadMasters: () => Promise<void>
  loadAppTasks: (userId: string) => Promise<void>
  loadBacklogToken: (userId: string) => Promise<void>
  incrementUsage: (taskId: number) => Promise<void>
}
```

### 2-2. 非同期アクション設計

#### `loadMasters()`
- `clients` を `deleted_at IS NULL` で全件取得
- `projects` を `deleted_at IS NULL` で全件取得（client_id でソート）
- `project_categories` を `deleted_at IS NULL` で全件取得
- 3クエリ並列実行（`Promise.all`）

#### `loadAppTasks(userId)`
- `app_tasks` を `user_id = userId AND deleted_at IS NULL` で取得
- `usage_count DESC` でソート（よく使うタスクを上位に）

#### `loadBacklogToken(userId)`
- `backlog_tokens` を `user_id = userId` で取得
- トークンが期限切れの場合はリフレッシュを試みる

#### `incrementUsage(taskId)`
- `app_tasks` の `usage_count` を +1（`usage_count + 1` で更新）

### 3-2. TodayView

**責務:** Googleカレンダー予定の取得・表示・タイマー制御。

**Props:** なし

**State:**
| 変数 | 型 | 説明 |
|------|----|------|
| loading | boolean | データ取得中フラグ |
| linkTarget | Event\|null | 紐付けモーダル対象イベント |
| todayRecord | Record\|null | 当日の app_records 行 |

**初期化フロー:**
```
1. Google Calendar API でカレンダー予定取得（fetchTodayEvents）
2. app_records で target_date の記録取得（なければ INSERT）
3. app_record_details で当日の明細取得
4. カレンダー予定と明細をマージ（calendarEventId で結合）
5. 未紐付けイベントに autoLink() を適用（confidence=high のみ自動設定）
6. todayEvents にセット
```

**イベント構造:**
```typescript
interface TodayEvent {
  id: string                    // calendar_event_id
  calendarEventTitle: string
  plannedStart: Date
  plannedEnd: Date
  detailId: number | null       // app_record_details.id
  taskId: number | null
  task: Task | null             // joined from appTasks
  actualStart: Date | null
  actualEnd: Date | null
  pauseLog: PauseEntry[]        // [{s: ISO, e: ISO}]
  overrideElapsedMs: number | null
  status: 'pending' | 'running' | 'done'
  autoLinked: boolean
}
```

**`handleStart(eventId)`:**
1. 既存の`app_record_details`があれば `actual_start` を更新、なければ INSERT
2. `activeEventId` をストアにセット
3. `isPaused = false` にセット

**`handleEnd(eventId)`:**
1. `actual_end = now()` で更新
2. `pause_log` を最終化（開いているポーズがあれば閉じる）
3. `activeEventId = null` にセット

**`handlePause()`:**
1. `isPaused = !isPaused` をセット
2. 停止時: `pausedAt = now()` をセット
3. 再開時: pause_log に `{s: pausedAt, e: now()}` を追加、DBに保存

### 3-7. AnalyticsView

**データ取得（2ステップ）:**

```javascript
// Step 1: 対象期間の app_records を user_id で絞り込み
const { data: recordRows } = await supabase
  .from('app_records')
  .select('id, target_date')
  .eq('user_id', session.user.id)
  .gte('target_date', sinceStr)

const recordIds = recordRows.map(r => r.id)
const recordMap = Object.fromEntries(recordRows.map(r => [r.id, r]))

// Step 2: record_id で app_record_details を取得（FK不要）
const { data: details } = await supabase
  .from('app_record_details')
  .select('*')
  .in('record_id', recordIds)
  .not('actual_end', 'is', null)

// Step 3: recordMap でエンリッチ
const enriched = details.map(r => ({
  ...r,
  app_records: recordMap[r.record_id] || null,
}))
```

**実績時間計算:**
```javascript
function netMinutes(r) {
  if (r.override_elapsed_ms != null) return r.override_elapsed_ms / 60000
  if (!r.actual_start || !r.actual_end) return 0
  const total = new Date(r.actual_end) - new Date(r.actual_start)
  const paused = (r.pause_log || []).reduce((acc, p) => {
    if (p.s && p.e) acc += new Date(p.e) - new Date(p.s)
    return acc
  }, 0)
  return Math.max(0, (total - paused) / 60000)
}
```

## 4. タイマーロジック

### 4-1. 実績時間計算（`calcNetElapsed`）

```javascript
function calcNetElapsed(event, isPaused, pausedAt) {
  if (!event.actualStart) return 0
  const now = Date.now()
  const total = now - new Date(event.actualStart).getTime()

  // 完了済みポーズの合計
  const pausedMs = (event.pauseLog || []).reduce((acc, p) => {
    if (p.s && p.e) acc += new Date(p.e) - new Date(p.s)
    return acc
  }, 0)

  // 現在進行中のポーズ
  const currentPauseMs = isPaused && pausedAt
    ? now - pausedAt.getTime()
    : 0

  return Math.max(0, total - pausedMs - currentPauseMs)
}
```

### 4-2. 表示用時間（`getDisplayElapsed`）

```javascript
function getDisplayElapsed(event, isPaused, pausedAt) {
  // 手動調整値を優先
  if (event.overrideElapsedMs != null) return event.overrideElapsedMs
  return calcNetElapsed(event, isPaused, pausedAt)
}
```

### 4-3. カウントダウン表示（`formatCountdown`）

```javascript
function formatCountdown(ms, withSign = false) {
  const abs = Math.abs(ms)
  const sign = ms < 0 && withSign ? '-' : ''
  const h = Math.floor(abs / 3600000)
  const m = Math.floor((abs % 3600000) / 60000)
  const s = Math.floor((abs % 60000) / 1000)
  if (h > 0) return `${sign}${h}:${pad(m)}:${pad(s)}`
  return `${sign}${pad(m)}:${pad(s)}`
}
```

## 5. 自動タスク紐付けロジック

### 5-1. スコアリングアルゴリズム（`autoLink.js`）

```javascript
function scoreCandidates(eventTitle, tasks) {
  const eventTokens = tokenize(eventTitle)

  return tasks
    .map(task => {
      const taskTokens = tokenize(task.title)
      const overlap = eventTokens.filter(t => taskTokens.includes(t)).length
      const titleScore = overlap / Math.max(eventTokens.length, taskTokens.length, 1)
      const historyBonus = Math.min(task.usage_count / 20, 0.3)
      const score = Math.min(1.0, titleScore + historyBonus)
      return { ...task, score }
    })
    .sort((a, b) => b.score - a.score)
}

function tokenize(str) {
  return str
    .toLowerCase()
    .split(/[\s\u3000\u30FB\uFF1A\uFF1F\u300C\u300D\u3010\u3011\uff08\uff09\u0028\u0029\u0020\u002F\u002E]+/)
    .filter(Boolean)
}
```

### 5-2. 信頼度レベル

| confidence | スコア範囲 | 動作 |
|-----------|---------|------|
| `high` | 0.65以上 | 自動紐付け実施 |
| `medium` | 0.35〜0.64 | LinkModalで提案（要確認） |
| `none` | 0.35未満 | 提案なし |

## 7. エラーハンドリング

### 7-1. Supabase クエリエラー

```javascript
const { data, error } = await supabase.from('app_tasks').select('*')
if (error) {
  console.error('Failed to fetch tasks:', error)
  // UIにはエラー表示しない（現状。今後改善予定）
```

### 7-2. Google Calendar API エラー

- `401 Unauthorized` → `provider_token` 期限切れ。再ログインを促す
- `403 Forbidden` → スコープ不足。再ログインを促す
- その他のエラー → コンソール出力のみ

### 7-3. Backlog API エラー

- `401 Unauthorized` → トークンリフレッシュを試みる
- リフレッシュ失敗 → BacklogModalでエラー表示
- その他のエラー → `console.error` でログ出力

- ユーザーへのトースト通知がない
- ネットワークエラー時のリトライロジックがない
