## 1. 概要

タスクタイマーは、Googleカレンダーの予定と連携して **「計画した時間を守る習慣」** を作るためのWebアプリです。

### 主な機能

| 機能 | 説明 |
|------|------|
| タイマー | Googleカレンダーの予定ごとに計測。残り時間・超過時間をリアルタイム表示 |
| タスク紐付け | カレンダー予定とタスクを紐付け。AI自動提案あり |
| 予実管理 | 計画時間と実績時間の差分を自動計算 |
| 集計・分析 | 日別・月別・タスク別・案件別・カテゴリ別に集計 |
| Backlog連携 | Backlogの課題をタスクとしてインポート |

## 2. ログイン

### 2-1. Googleアカウントでログイン（通常）

1. ブラウザで本アプリのURLを開く
2. **「Google アカウントでログイン」** ボタンをクリック
3. Googleアカウント選択画面が表示されるので、使用するアカウントを選択
4. カレンダーへのアクセス許可を求められたら **「許可」** をクリック
5. アプリのメイン画面（今日タブ）が表示される

> **注意:** カレンダーの読み書き権限が必要です。許可しないと今日タブが動作しません。

### 2-2. メールでログイン（開発環境のみ）

開発環境（DEVバッジが表示されている場合）では、メールアドレスとパスワードでもログインできます。

1. 「DEV MODE」セクションのEmail欄にメールアドレスを入力
2. Password欄にパスワードを入力
3. **「メールでログイン」** をクリック

### 2-3. ログアウト

1. 右上のユーザー名（アバター）をクリック
2. ログアウトボタンをクリック

## 4. 今日タブ — タイマーを使う

### 4-1. 画面構成

```
┌─────────────────────────────────────────────┐
│  タイマーヒーロー（アクティブなタスクの表示）          │
├─────────────────────────────────────────────┤
│  操作ボタン（開始・一時停止・終了）                  │
├─────────────────────────────────────────────┤
│  本日のスケジュール  X/Y 完了                    │
│  ┌─────────────────────────────────────────┐ │
│  │ TaskCard（カレンダー予定ごとに1枚）           │ │
│  │ TaskCard                                │ │
│  └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

### 4-2. タスクカードの見方

```
┌──────────────────────────────────────────────────┐
│ 09:00–10:00  [未着手]                             │
│ MTG：週次定例                                      │
│ [#42 基幹システム改修]  [● ITreat]                  │
│ 経過: --                          [開始]           │
└──────────────────────────────────────────────────┘
```

| 要素 | 説明 |
|------|------|
| 時刻 | カレンダーの予定開始〜終了時刻 |
| ステータスバッジ | 未着手 / 進行中 / 一時停止中 / 完了 |
| タスクID | 紐付けられたタスク（未紐付けの場合は「未紐付け ＋」） |
| クライアント名 | クライアントカラーと名前 |
| 経過時間 | 実際に計測した時間 |

### 4-3. タイマーの開始

1. 開始したいタスクカードの **「開始」** ボタンをクリック
2. タイマーヒーローエリアに現在のタスク情報が表示される
3. カウントダウン（残り時間）がリアルタイムで更新される

### 4-4. タイマーのステータス表示

| 状態 | 色 | 説明 |
|------|----|------|
| 進行中 | 青 | 残り時間がある（正常） |
| 一時停止中 | 黄 | ⏸ 一時停止中 |
| 超過中 | 赤 | ⚠ 計画時間を超過 |

### 4-5. 一時停止・再開

- **「⏸ 一時停止」** ボタンで一時停止（停止中の時間は実績から除外される）
- 一時停止中は **「▶ 再開」** ボタンで再開

### 4-6. 時間の手動調整

タイマー進行中、スライダーで実績時間を手動調整できます。

1. スライダーを左右にドラッグ（0分〜計画時間×2まで）
2. 「✎」マークが表示されたら手動調整済み
3. **「リセット」** ボタンで自動計測に戻す

### 4-7. タイマーの終了

- **「終了」** ボタンをクリック
- タスクカードのステータスが「完了」に変わり、実績時間が表示される

### 4-8. やり直し（Undo）

終了後にやり直したい場合：

1. 完了したタスクカードの **「↩ やり直し」** ボタンをクリック
2. タスクのステータスが「未着手」に戻る

### 4-9. タスクの紐付け

カレンダー予定とタスクを紐付けることで、集計が案件・カテゴリ別に分類されます。

**自動紐付け:**
予定タイトルとタスク名の類似度が高い場合、自動的に紐付けられます。

**手動紐付け:**

1. タスクカードの **「未紐付け ＋」** または **「#ID タスク名」** をクリック
2. 紐付けモーダルが開く
3. 提案されたタスクから選ぶ、またはテキストで検索
4. 確認ボタンをクリック

**紐付けの提案スコア:**

| アイコン | スコア | 意味 |
|---------|--------|------|
| 🟢 高 | 0.65以上 | タイトルが高い確率で一致 |
| 🟡 中 | 0.35〜0.64 | 部分的に一致 |
| ⚪ 低 | 0.35未満 | 一致度が低い |

## 5. タスク管理タブ — タスクを管理する

### 5-1. タスク一覧

- 全タスクが一覧表示される
- フィルター: クライアント別・ステータス別・繰り返しタスクのみ
- 期日超過のタスクは赤でハイライト表示

### 5-2. 新規タスク作成

1. **「＋ 新規タスク」** ボタンをクリック
2. 以下の情報を入力:

| 項目 | 必須 | 説明 |
|------|------|------|
| タスク名 | ◎ | タスクの名称 |
| ステータス | ◎ | 未着手 / 進行中 / 完了 |
| クライアント | △ | 担当クライアント |
| 案件 | △ | クライアント選択後に選択可 |
| カテゴリ（第1階層） | △ | 案件選択後に選択可 |
| カテゴリ（第2階層） | △ | カテゴリ選択後に選択可 |
| 開始日 | △ | タスクの開始予定日 |
| 期日 | △ | タスクの締め切り日 |
| 繰り返し | △ | 定期的なタスクの場合にチェック |

3. **「保存」** をクリック

### 5-3. タスク編集

1. 編集したいタスクの行の **「✏」** ボタンをクリック
2. 編集モーダルが開く
3. 変更後 **「保存」** をクリック

> **Backlogタスクの注意:** Backlogから取り込んだタスクはタイトルとステータスが読み取り専用です。

### 5-4. タスク削除

1. 削除したいタスクの **「🗑」** ボタンをクリック
2. 確認後、ソフトデリート（論理削除）される

### 5-5. Backlogバッジ

BacklogからインポートされたタスクはタスクIDの左に **緑の「B」バッジ** が表示されます。

## 6. 集計・履歴タブ — 実績を確認する

### 6-1. サブタブ

| タブ | 内容 |
|------|------|
| 日別 | 日ごとの作業記録。予定時間・実績時間・差分 |
| 月別 | 月ごとの集計 |
| タスク別 | タスクごとの効率・超過時間 |
| 案件別 | 案件（プロジェクト）ごとの集計 |
| カテゴリ別 | カテゴリ階層ごとの集計 |

### 6-2. 共通フィルター

- **期間:** 直近7日・14日・30日・90日・365日
- **クライアント:** 全クライアント / 個別選択

### 6-3. 指標の見方

| 指標 | 説明 |
|------|------|
| 記録日数 | 作業記録がある日数 |
| 総超過時間 | 計画を超えた時間の合計（分） |
| 全体効率 | 実績÷計画×100（%）。100%以下が良好 |

### 6-4. 効率の色分け

| 色 | 効率 | 意味 |
|----|------|------|
| 🟢 緑 | 100%未満 | 計画内で完了 |
| 🟡 橙 | 100〜110% | わずかに超過 |
| 🔴 赤 | 110%超 | 大幅超過 |

## 7. Backlog連携

### 7-1. 連携設定

1. ヘッダーの **「Backlog」** ボタンをクリック
2. **スペースキー** を入力（例: `your-company`）
3. **「連携する」** をクリック
4. Backlogの認証画面でログイン・許可
5. アプリに戻ったら完了（ボタンが緑色になる）

### 7-2. 課題のインポート

1. **「Backlog」** ボタンをクリック → **「課題をインポート」** タブを選択
2. 自分に割り当てられた未完了の課題が一覧表示される
3. インポートしたい課題にチェックを入れる
4. **「インポート」** をクリック
5. タスク管理タブに新しいタスクが追加される

### 7-3. 連携解除

1. **「Backlog」** ボタンをクリック
2. **「連携解除」** をクリック

## 8. よくある質問

**Q: 今日タブに予定が表示されない**
A: Googleカレンダーのアクセス権限が必要です。一度ログアウトして、再ログイン時に権限を許可してください。

**Q: タイマーを開始したら画面が白くなった**
A: ブラウザを再読み込みしてください。再現する場合は開発者にお知らせください。

**Q: Backlogの課題が表示されない**
A: 自分に担当者として割り当てられ、ステータスが「未対応」「処理中」「処理済み」の課題のみ表示されます。

**Q: 集計タブに何も表示されない**
A: タイマーを使って「終了」した記録がある日のデータのみ集計されます。まず今日タブでタイマーを使ってみてください。

**Q: 手動調整した時間を元に戻したい**
A: タイマーコントロールの「リセット」ボタンをクリックしてください。
